# YOLO26n 正式基线实验结果

## 实验配置

| 项目 | 设置 |
|---|---|
| 模型 | YOLO26n 预训练模型 |
| 参数量 | 约237.62万（融合后） |
| 计算量 | 约5.2 GFLOPs（融合后） |
| 输入尺寸 | 640×640 |
| Batch size | 16 |
| 最大轮数 | 100 |
| 早停 patience | 20 |
| 实际训练轮数 | 89 |
| 最佳轮次 | 69 |
| 优化器 | AdamW（Ultralytics auto） |
| AMP | 开启 |
| 随机种子 | 42 |
| GPU | NVIDIA GeForce RTX 3080 10GB |

训练约耗时278.45秒。PyTorch记录的峰值已分配显存约2.70GiB，训练器峰值显示约4.62GB。训练在第89轮触发早停，最佳模型来自第69轮。

## 验证集结果

最佳权重在59张验证图片、1178个标注框上的总体结果：

| Precision | Recall | mAP50 | mAP50-95 |
|---:|---:|---:|---:|
| 0.7417 | 0.7114 | 0.7860 | 0.5754 |

第69轮记录的训练框损失为1.03557、训练分类损失为0.82042，验证框损失为1.03318、验证分类损失为0.88008。训练和验证损失走势接近，未出现明显的严重过拟合，但第69轮后综合指标进入平台期并轻微波动，因此早停选择合理。

## 独立测试集结果

模型选择完全依据验证集完成后，固定 `best.pt` 在60张独立测试图片、1274个标注框上评价一次。

| Precision | Recall | mAP50 | mAP50-95 |
|---:|---:|---:|---:|
| 0.6944 | 0.7796 | 0.7711 | 0.5667 |

测试集 mAP50-95 比验证集低约0.0087，差距较小，说明当前场景分组划分下模型总体泛化结果稳定。

## 各类别测试结果

| 类别 | Precision | Recall | AP50 | AP50-95 |
|---|---:|---:|---:|---:|
| 书写 | 0.6379 | 0.8667 | 0.8273 | 0.5893 |
| 阅读 | 0.5827 | 0.4993 | 0.5543 | 0.3883 |
| 抬头听讲 | 0.7177 | 0.8819 | 0.8484 | 0.6044 |
| 转头 | 0.5772 | 0.6173 | 0.6044 | 0.4405 |
| 举手 | 0.7263 | 0.7319 | 0.7660 | 0.5466 |
| 站立 | 0.8866 | 0.8889 | 0.8643 | 0.6531 |
| 讨论 | 0.7324 | 0.9710 | 0.9329 | 0.7446 |

阅读和转头是基线模型的主要弱项。混淆矩阵显示部分阅读目标被识别为抬头听讲，转头也会与抬头听讲互相混淆，这与三类行为在远距离课堂画面中的姿态差异较细微有关。

站立和讨论取得较高AP，但测试集分别只有9个和31个标注框，置信区间会明显宽于其他类别，不能仅凭这一结果宣称少数类问题已经解决。讨论类别通常框选整个小组，目标尺寸较大且场景特征明显，也会使其检测难度低于单个小目标行为。

## 推理性能

在RTX 3080、640输入、batch=16测试评估条件下，每张图片平均：

- 预处理：0.99ms；
- 模型推理：9.37ms；
- 后处理：1.19ms；
- 合计约11.55ms，不计文件读取与界面渲染。

仅按模型推理时间换算约为106.7 FPS；按预处理、推理和后处理总时间换算约为86.6 FPS。该结果来自批量验证，最终系统的单张图片和视频端到端速度需要在系统测试阶段重新测量。

## 基线结论

YOLO26n以较小模型规模取得0.7711的测试mAP50和0.5667的测试mAP50-95，能够作为课程设计的有效基线。下一阶段应重点分析：

1. 阅读、转头与抬头听讲之间的细粒度混淆；
2. 小目标漏检和背景误检；
3. 类别长尾对站立、讨论指标稳定性的影响；
4. 输入分辨率、模型规模和针对性增强的改进潜力。
