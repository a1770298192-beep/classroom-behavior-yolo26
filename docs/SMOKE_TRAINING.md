# YOLO26n 冒烟训练验收记录

## 目的

本次运行仅验证数据读取、CUDA、自动混合精度、训练/验证/测试流程、权重保存和日志生成，不作为正式模型性能结论。

## 配置

| 项目 | 设置 |
|---|---|
| 模型 | YOLO26n，ImageNet/COCO预训练权重 |
| 输入尺寸 | 640×640 |
| 训练轮数 | 3 |
| 批大小 | 8 |
| 优化器 | Ultralytics auto，实际选择 AdamW |
| 初始学习率 | 0.000909（auto结果） |
| AMP | 开启 |
| 随机种子 | 42 |
| 设备 | NVIDIA GeForce RTX 3080 10GB |

## 训练链路结果

- 281张训练图片和59张验证图片全部扫描成功，无背景空图和损坏文件。
- AMP自检通过。
- 3轮训练约耗时58.65秒。
- PyTorch记录的峰值已分配显存约1.41GiB，训练器显示约1.95GB GPU内存。
- `best.pt` 和 `last.pt` 均成功保存并可重新加载，单个权重约5.36MB。
- 训练配置、CSV日志、训练曲线、PR曲线、混淆矩阵、批次增强图和验证预测图均生成成功。
- 统一训练脚本提供 `--resume` 参数，可在训练意外中断且保留训练状态的 `last.pt` 上恢复；已正常完成的冒烟权重在收尾阶段被Ultralytics剥离优化器状态，属于推理/评估权重，不用于验证“已完成训练再次续跑”。

分类损失持续下降，框回归损失总体下降，验证mAP持续上升，说明梯度更新、数据加载和评价流程正常。第3轮仍处于预热结束附近，低精度不能用于评价模型上限。

独立测试集管线也已验证成功，3轮模型在60张测试图片上的 mAP50 为0.11642，mAP50-95为0.06767。该数字仅作为管线通畅的证据。

## 数据增强检查

训练批次图显示 mosaic、随机缩放、平移和水平翻转能够正确组合图片，标注框随增强同步变化，未发现明显丢框或错位。由于课堂画面包含大量密集目标，增强后单批实例数较高，但未发生显存溢出。

## 正式训练建议

RTX 3080在batch=8时显存余量较大。正式基线建议采用batch=16、100轮、640输入和早停策略。batch=16预计仍有充足显存，同时能减少每轮迭代次数；若遇到偶发峰值再回退至batch=12或8。
